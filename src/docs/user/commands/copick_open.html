Command: copick open, show, hide

Usage:
copick open|show|hide picks|mesh|segmentation [ uri ]

These commands show or hide picks, meshes and segmentations of the active run in the Copick tool. They are the command equivalents of toggling an entity's visibility in the Picks / Meshes / Segmentations tabs.

  • copick open and copick show are equivalent: each loads the entity if needed and ensures it is visible (so show also re-displays a hidden entity).
  • copick hide hides a loaded entity.

The optional uri is a copick URI selecting which entities to act on. URIs support glob (default) and re: regex patterns, so one command can act on several entities; the number affected is reported in the Log. When uri is omitted, the command acts on all entities of that type in the active run.

entityURI scheme
picksobject_name:user_id/session_id
meshobject_name:user_id/session_id
segmentation name:user_id/session_id@voxel_size[?multilabel=true]

A run must be open first (see copick open run); otherwise the command reports that no run is active. Quote URIs that contain wildcards so they are not split by the command line, e.g. "ribosome:*/*".

Examples:

copick open picks
copick open picks ribosome:alice/0
copick show picks "ribosome:*/*"
copick hide picks ribosome:alice/0
copick open mesh membrane:bob/12
copick open segmentation painting:alice/3@10.0

See also: Copick URIs, copick open run, copick new picks.
