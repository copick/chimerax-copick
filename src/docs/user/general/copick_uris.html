Copick URIs

Several copick commands select existing entities (picks, meshes, segmentations) by a copick URI. A URI is a compact, relative identifier – it is interpreted against the currently loaded project and (for the visualization commands) the currently active run.

URI schemes

entity URI scheme example
picks object_name:user_id/session_id ribosome:alice/0
mesh object_name:user_id/session_id membrane:bob/12
segmentation name:user_id/session_id@voxel_size[?multilabel=true] painting:alice/3@10.0
tomogram tomo_type@voxel_size wbp@10.0

Patterns

URIs support matching, so a single command can act on several entities at once:

  • Glob (the default): use * and ? wildcards, e.g. ribosome:*/* matches every ribosome pick set in the active run.
  • Regex: prefix the URI with re:, e.g. re:ribo.*:.*/\d+.
  • Incomplete URIs auto-complete missing components to wildcards, e.g. ribosome is treated as ribosome:*/*.

When a URI is omitted entirely, the command matches all entities of that type in the active run. Because :, /, @ and ? are part of the URI token, quote the URI if your shell or the command line would otherwise split it, e.g. "ribosome:*/*".

Note: a session_id of 0 denotes tool-generated entities, which are read-only.
